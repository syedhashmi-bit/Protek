Publish Protek's intel as a signed feed (phase 78)

When enabled, your locally-detected decisions are exposed as a public Ed25519-signed JSON feed at /feed/banned-ips.signed.json. Other Protek operators can subscribe to it (add as a CrowdSec source on their end) and verify each fetch with the public key at /feed/pubkey. Community-blocklist passthroughs (origin lists:*) are excluded by default — you republish your own intel, not other people's.

Subscriber example

import base64, json
import requests
from nacl.signing import VerifyKey                   # `pip install pynacl`

PUB_B64 = "{{ status.pub_key_b64 or '' }}"
verifier = VerifyKey(base64.b64decode(PUB_B64))

r = requests.get("https://protek.example.com/feed/banned-ips.signed.json"
                 "?subscriber=my-other-protek").json()
canonical = json.dumps(r["body"], sort_keys=True, separators=(",", ":")).encode()
verifier.verify(canonical, base64.b64decode(r["signature"]))
print("verified seq=" + str(r["body"]["sequence"]),
      "count=" + str(r["body"]["count"]))
