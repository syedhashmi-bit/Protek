Add federation source

A federation source is a remote CrowdSec LAPI whose decisions Protek will pull, dedupe, and push to every configured bouncer. The wizard walks you through pointing Protek at a remote LAPI and verifying the handshake works before the source goes live. Skip the wizard if you already know all the values.

The remote CrowdSec LAPI's base URL. Transport is your choice — WireGuard / Tailscale private IP, or a public TLS endpoint. Do not include a trailing slash.

{% endcall %} {% call wizard_step(2, "Run on remote") %}

Run these commands on the remote VPS (whose LAPI you entered above). They register Protek as a bouncer on that side and print a bouncer key you'll paste in the next step. The commands are parameterized — the placeholder <this-host> should be replaced with a recognizable label (e.g. your VPS A's hostname). 

# 1. Install CrowdSec on the remote if it isn't already.
curl -s https://install.crowdsec.net | sudo sh
sudo apt install -y crowdsec
sudo systemctl enable --now crowdsec

# 2. Register Protek as a bouncer and capture the printed key.
sudo cscli bouncers add protek-from-<this-host>
# → copy the printed key. You will NOT see it again.

# 3. Allow the remote LAPI's port (default 8080) from this Protek host's
#    WG / private IP. Adjust to your firewall:
sudo ufw allow from {{ protek_host_ip or '<protek-host-IP>' }} to any port 8080 proto tcp

# 4. Verify the LAPI binds on the right interface (default 127.0.0.1).
#    Edit /etc/crowdsec/config.yaml if you need it on a private IP:
#       api: { server: { listen_uri: :8080 } }
#    Then restart: sudo systemctl restart crowdsec

When the cscli bouncers add command completes, you'll see a 40-char hex key. Continue to the next step to paste it.

{% endcall %} {% call wizard_step(3, "Paste key") %}

Stored at rest; never displayed back in the UI once saved. Rotate with cscli bouncers add on the remote and update the source row from /federation.

{% endcall %} {% call wizard_step(4, "Test + save") %}

The form runs a connection test against the remote LAPI before saving (HTTP fetch + auth handshake + version probe). If the test fails, the wizard returns you to step 1 with the error displayed so you can correct the URL or key without re-typing.

On success: the source row is created, bootstrap pulls active decisions on the next reconcile cycle (default 10 s), and the audit log records who added it.
