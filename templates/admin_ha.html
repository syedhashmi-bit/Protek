Manual failover runbook

  1. Confirm the primary is down — curl https://<primary>/health returns connection-refused or persistent 503 with poll_stale.
  2. Stop Litestream on the standby — systemctl stop litestream — so it doesn't keep overwriting the DB while you promote.
  3. Promote here — use the form above. Audit row is written + flag flipped.
  4. Restart Protek on this host — systemctl restart protek. The boot-time gate now starts the poller because ha.role=primary.
  5. Update DNS if the public hostname needs to point at this host. Caddy/nginx on either host will keep serving its local Protek; the rest is operator policy.
  6. When the original primary returns, demote it via its own /admin/ha, restart Protek, then start Litestream pointing at this host as the new source-of-truth. Re-baseline if the chain diverged (see docs/DR-RUNBOOK.md §3).

Auto-failover is deliberately not enabled — split-brain protection without a real consensus layer is hard, and a partition where the primary is alive but unreachable from the standby would otherwise produce two primaries racing the MT. This phase ships the scaffolding; automation is 2.1+.
