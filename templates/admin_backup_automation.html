BACKUP_PASSPHRASE missing

Automation will fail until BACKUP_PASSPHRASE is set in .env. Pick a long random passphrase (≥12 chars) and store it somewhere safe — losing it makes the bundles unreadable. 

echo "BACKUP_PASSPHRASE=$(openssl rand -base64 32)" >> /var/www/Protek/.env
systemctl restart protek

S3 / B2 credentials (read from .env)

For Backblaze B2: set BACKUP_S3_ENDPOINT to your bucket's S3 endpoint (e.g. https://s3.us-west-002.backblazeb2.com), region to the matching value (e.g. us-west-002), and use the bucket-scoped Application Key ID + Application Key as BACKUP_S3_KEY / BACKUP_S3_SECRET. For AWS S3: omit the endpoint, set the region, use IAM access keys.

▸ How to install Litestream 
# amd64 (most x86 VPS)
curl -L https://github.com/benbjohnson/litestream/releases/latest/download/litestream-linux-amd64.tar.gz \
  | tar -xz -C /usr/local/bin litestream

# arm64 (Hetzner CAX, AWS Graviton, etc.)
curl -L https://github.com/benbjohnson/litestream/releases/latest/download/litestream-linux-arm64.tar.gz \
  | tar -xz -C /usr/local/bin litestream

# Copy scaffolds Protek ships:
cp {{ litestream.sample_config }} /etc/litestream.yml
cp {{ litestream.sample_unit }} /etc/systemd/system/litestream.service

# Edit /etc/litestream.yml — paste your B2 bucket / keys (same values as
# the BACKUP_S3_* env vars).

systemctl daemon-reload
systemctl enable --now litestream
journalctl -u litestream -f
▸ Restore from Litestream 
systemctl stop protek litestream
litestream restore -o /var/www/Protek/protek.db /var/www/Protek/protek.db
systemctl start litestream protek
