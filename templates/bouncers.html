Config shapes

mikrotik        →  {"host":"10.0.0.1", "username":"protek-api", "password":"<...>",
                    "port":8728, "use_ssl":false,
                    "address_list":"crowdsec"}
                   # additional router beyond the .env-anchored one.
                   # operator owns the firewall drop rule on the router.
                   # optional filters: "max_entries":30000, "exclude_origins":["lists:*"]

iptables_ipset  →  {} (no config — uses default set names protek-bans / protek-bans6)
                   optional: {"set_v4":"my-set","set_v6":"my-set6","max_v4":200000}

cloudflare      →  {"account_id":"<cf-acct-id>", "api_token":"<scoped-token>",
                    "list_name":"protek_bans"}    # auto-creates list
                   optional filters: "max_entries":9500, "exclude_origins":["lists:firehol_cruzit_web_attacks"]

pfsense         →  {"base_url":"https://pf.example.com",
                    "api_key":"<X-API-Key>",
                    "alias_name":"protek_bans",
                    "verify_tls": false}

opnsense        →  {"base_url":"https://opn.example.com",
                    "api_key":"<key>", "api_secret":"<secret>",
                    "alias_name":"protek_bans",
                    "verify_tls": false}

ALL ADAPTERS support optional per-bouncer filter knobs:
  "origins":         ["crowdsec", "lists:firehol_greensnow"]  (whitelist)
  "exclude_origins": ["lists:*"]                              (blacklist)
  "max_entries":     9500                                     (newest-first cap)
